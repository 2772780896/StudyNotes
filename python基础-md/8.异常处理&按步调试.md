## 异常处理

### try / except / else

```python
try:
    test_statement
except error_type:
    statement_of_reporting_error
else:
    statement_while_no_error
```

执行逻辑：

- `try` 块正常运行
- 如果发生异常且类型匹配 `error_type` → 执行 `except` 块
- 如果发生异常但类型不匹配 → 终止并向上一层报错
- 如果没有异常 → 执行 `else` 块

### finally

```python
try:
    test_statement
finally:
    statement_whatever_if_error
```

无论 `try` 是否报错，`finally` 块都会执行。

> 注意：`try` 框架之外的语句，在 `try` 未报错或异常被捕获后，仍会继续执行。

## pdb 终端调试

```bash
python -m pdb 文件名
```

进入 pdb 界面后的操作命令：

| 命令 | 说明 |
|------|------|
| `n` | 单步执行代码 |
| `p 变量名` | 查看变量值 |
| `q` | 退出 pdb |

### 指定断点

```python
import pdb

pdb.set_trace()     # 在此处暂停，进入调试模式
```

设置断点后直接运行文件即可，无需 `-m pdb`。

## IDE 调试（Debug）

在 PyCharm 中，点击代码行号左侧标记红点（断点），然后以 Debug 模式运行，代码将在断点处暂停。

可以设置多个断点，按步执行（Step Over / Step Into / Step Out）来逐步排查问题。
