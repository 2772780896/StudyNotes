## PEP8 编码规范

### 编码

- 缩进固定为 **4 个空格**（一个 Tab）
- 单行字符数不超过 **79**
- 运算符（`=`、`==`、`<`、`>`、`!=`）两侧各加一个空格。但用于关键字参数或默认参数时例外：

  ```python
  def func(a=1):    # 正确：= 两侧不加空格
      pass
  ```

- 顶级函数之间空两行，类中方法之间空一行
- 注释 `#` 后空一格再写内容
- 使用块注释解释一段代码的逻辑

### 命名

- 非 lambda 中避免单字母变量名

  ```python
  f = lambda x: x + 1    # lambda 中可用单字母
  ```

- 函数名、模块名全小写，单词用下划线连接
- 类名、异常名使用驼峰（CamelCase）
- 全局变量尽量全大写，单词用下划线连接
- 函数名必须包含动词

### 导入

- 禁止 `from package import *`
- 顺序：标准库 → 第三方库 → 自定义库
- 单行不导入多个库

### 字符串拼接

字符串拼接用 `join()` 比 `+` 效率更高。

```python
''.join([a, b, c])    # 推荐
a + b + c              # 效率低
```

## 检查 PEP8 规范

安装 `pycodestyle` 或 `pylint` 后在终端执行：

```bash
pycodestyle python文件
pylint python文件
```
