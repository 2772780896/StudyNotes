# Mockjs 模拟数据

Mockjs 是一款前端模拟数据生成器，允许前端在后端接口未就绪时独立开发。

## 1. 核心原理

Mockjs 拦截了浏览器的 `XMLHttpRequest` 请求。当请求的 URL 匹配 Mock 定义的规则时，它会返回模拟数据而非发送真实网络请求。
> **注意**：在浏览器的 Network 面板中通常看不到这些拦截的请求。

## 2. 快速上手

- **安装**：`npm i mockjs`
- **引入**：在入口文件（如 `main.js` 或 `index.js`）中导入。
  ```javascript
  import './mock'; // 假设你的 mock 逻辑在 mock 目录下
  ```

## 3. 数据模板定义

```javascript
Mock.mock('/api/user', 'get', {
  code: 200,
  message: 'success',
  'list|1-10': [{ // 随机生成 1-10 个元素的数组
    'id|+1': 1, // id 自增
    name: '@cname', // 随机中文名
    'age|18-60': 1, // 18-60 之间的整数
    'score|1-100.1-2': 1, // 浮点数
    city: '@city', // 随机城市
    avatar: "@image('100x100', '#50B347', '#FFF', 'Mock.js')" // 随机图片
  }]
});
```

## 4. 常用占位符 (Random 语法)

| 占位符 | 描述 | 示例 |
| :--- | :--- | :--- |
| `@cname` | 随机中文名 | `张三` |
| `@cword(min, max)` | 随机中文字符 | `你好` |
| `@email` | 随机邮箱 | `test@163.com` |
| `@date` | 随机日期 | `2023-10-01` |
| `@time` | 随机时间 | `14:30:00` |
| `@color` | 随机颜色 | `#3538B2` |
| `@pick(list)` | 从数组中随机选一个 | `Mock.Random.pick([1, 2, 3])` |

## 5. 处理带参数的请求

### 5.1 GET 请求参数
```javascript
Mock.mock(/\/api\/detail\?id=\d+/, 'get', (options) => {
  const url = new URL(options.url, 'http://localhost');
  const id = url.searchParams.get('id');
  return { id, data: '...' };
});
```

### 5.2 POST 请求参数
```javascript
Mock.mock('/api/login', 'post', (options) => {
  const body = JSON.parse(options.body); // 获取 POST 参数
  return { code: 200, user: body.username };
});
```

---
**复习提示**：Mockjs 适合快速原型开发。对于更真实的网络拦截（如能在 Network 面板看到），现代开发更推荐使用 **MSW (Mock Service Worker)**。
