# 20. React 进阶全攻略

本章节涵盖了 React 生态中的核心插件：状态管理、路由导航以及 UI 组件库。

## 1. Redux：集中状态管理

Redux 用于管理多个组件共享的数据状态。

### 1.1 核心概念
- **Reducer**：一个纯函数，接收旧 `state` 和 `action`，返回新 `state`。
  ```javascript
  function reducer(state = { count: 0 }, action) {
    if (action.type === 'ADD') return { count: state.count + 1 };
    return state;
  }
  ```
- **Store**：存储状态的容器。`const store = createStore(reducer);`
- **Dispatch**：提交修改。`store.dispatch({ type: 'ADD' });`
- **getState**：获取当前状态。`store.getState();`
- **Subscribe**：监听变化。`store.subscribe(() => console.log('变了'));`

### 1.2 Redux Toolkit (RTK) - 官方推荐方式
RTK 极大地简化了 Redux 的代码量。
- **模块化结构**：
  - `src/store/modules/xxxStore.js`：定义切片 (Slice)。
  - `src/store/index.js`：组合模块并导出 store。
- **创建切片 (createSlice)**：
  ```javascript
  const userStore = createSlice({
    name: 'user',
    initialState: { token: '' },
    reducers: {
      setToken(state, action) {
        state.token = action.payload; // RTK 内部支持直接修改写法
      }
    }
  });
  ```
- **异步处理**：在模块中定义并导出异步函数（Thunk），内部调用 `dispatch`。

### 1.3 React-Redux 连接
- **Provider**：在根组件包裹 `<Provider store={store}>`。
- **Hooks**：
  - `useSelector`：获取状态。`const token = useSelector(state => state.user.token);`
  - `useDispatch`：发送 Action。`const dispatch = useDispatch();`

## 2. React Router：前端路由

用于根据 URL 的变化渲染不同的组件，实现单页应用 (SPA)。

### 2.1 路由模式
- **BrowserRouter**：使用 HTML5 历史记录（推荐，URL 正常）。
- **HashRouter**：使用 URL 哈希值（带有 `/#/`）。

### 2.2 基础配置
- **创建路由对象**：
  ```javascript
  const router = createBrowserRouter([
    { path: '/home', element: <Home /> },
    { path: '/about/:id', element: <About /> } // 动态路由
  ]);
  ```
- **应用路由**：`<RouterProvider router={router} />`。

### 2.3 导航方式
- **声明式导航**：`<Link to="/path">跳转</Link>`。
  - 优于 `<a>` 标签，因为它不会刷新整个页面，保持了应用状态。
- **编程式导航**：使用 `useNavigate` 钩子。
  - `navigate('/path')`
  - `navigate('/path', { state: { id: 1 } })` (传状态)。

### 2.4 参数获取
- **SearchParams**：`?id=1&name=xx`。使用 `useSearchParams()` 获取。
- **Params**：`/path/:id`。使用 `useParams()` 获取。

### 2.5 嵌套路由 (二级路由)
- **配置**：在父路由下增加 `children` 数组。
- **渲染位置**：父组件中使用 `<Outlet />` 指明子组件的渲染位置。
- **默认渲染**：设置 `index: true`。

### 2.6 高级功能
- **404 页面**：设置 `path: '*'`。
- **路由懒加载**：使用 `lazy(() => import(...))` 和 `<Suspense>` 包裹，实现按需加载。

## 3. Ant Design (antd)：UI 组件库

Ant Design 是 React 社区最成熟的 UI 组件库之一。

### 3.1 常用组件
- **Button**, **Card**：基础布局组件。
- **Form (表单)**：
  - **校验**：通过 `rules` 属性定义规则。
  - **提交**：`onFinish` 会在校验通过后执行，并接收表单数据对象。
  - **数据回填**：通过 `Form.useForm()` 获取 form 实例，调用 `form.setFieldsValue(data)`。
- **Upload (上传)**：
  - `onChange`：获取上传进度和文件状态。
  - `action`：指定上传接口。默认发送 `multipart/form-data`。
