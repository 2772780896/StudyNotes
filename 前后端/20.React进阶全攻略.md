## 1. Redux：集中状态管理

Redux 用于管理跨组件的数据状态，解决多层级组件通信难题。

### 1.1 原生 Redux 核心流程
- **定义 Reducer 函数**：
  ```javascript
  function reducer(state = { count: 0 }, action) {
    if (action.type === 'increment') {
      return { count: state.count + 1 };
    }
    return state;
  }
  ```
- **生成 Store 实例**：`const store = createStore(reducer);`
- **提交 Action**：`store.dispatch({ type: 'increment', data: 1 });`
- **获取 State**：`store.getState().count;`
- **监听变化**：`store.subscribe(() => { ... });`

### 1.2 Redux Toolkit (RTK) - 现代标准写法
`npm i @reduxjs/toolkit`
- **模块化 Slice (切片)**：
  ```javascript
  import { createSlice } from "@reduxjs/toolkit";
  const counterStore = createSlice({
    name: 'counter',
    initialState: { count: 0 },
    reducers: {
      increment(state, action) {
        state.count += action.payload; // RTK 内部集成 Immer，支持“直接修改”
      }
    }
  });
  export const { increment } = counterStore.actions;
  export default counterStore.reducer;
  ```
- **异步 Action (Thunk)**：
  ```javascript
  export const fetchCount = () => {
    return async (dispatch) => {
      const res = await axios.get('/api/count');
      dispatch(increment(res.data));
    };
  };
  ```
- **组合 Store**：使用 `configureStore` 组合多个模块的 reducer。

### 1.3 React-Redux：两者间的桥梁
`npm i react-redux`
- **注入 Store**：在入口 `index.js` 中使用 `<Provider store={store}>` 包裹 `<App />`。
- **组件中使用**：
  - `useSelector`：获取状态。`const count = useSelector(state => state.counter.count);`
  - `useDispatch`：获取派发函数。`const dispatch = useDispatch();`

## 2. React Router：前端路由管理

`npm i react-router-dom`

### 2.1 基础配置与路由模式
- **BrowserRouter**：基于 HTML5 History API，URL 正常（推荐）。
- **HashRouter**：基于 URL 哈希值（带有 `/#/`），兼容性好但 URL 略显复杂。
- **配置示例**：
  ```javascript
  const router = createBrowserRouter([
    { path: '/login', element: <Login /> },
    { 
      path: '/', 
      element: <Layout />, 
      children: [
        { index: true, element: <Home /> }, // 默认渲染
        { path: 'about/:id', element: <About /> } // 动态路由
      ] 
    }
  ]);
  ```

### 2.2 导航与跳转
- **声明式导航**：`<Link to="/path">跳转</Link>`。
  - 避免使用 `<a>` 标签，因为它会引起整页刷新导致 React 状态丢失。
- **编程式导航**：使用 `useNavigate()`。
  - `navigate('/path')` 或 `navigate('/path', { state: { id: 1 } })`。

### 2.3 参数获取
- **SearchParams**：`?id=1`。使用 `const [params] = useSearchParams(); params.get('id');`
- **Params**：`/:id`。使用 `const { id } = useParams();`

### 2.4 二级路由与 Outlet
- 在父路由组件中使用 `<Outlet />` 作为占位符，子路由组件会渲染在该位置。

### 2.5 路由懒加载
使用 `lazy` 和 `Suspense` 实现按需加载，优化首屏速度。
```jsx
const Home = lazy(() => import('./pages/Home'));
<Suspense fallback="加载中..."><Home /></Suspense>
```

## 3. Ant Design (antd)：UI 组件库

`npm i antd --save`

- **常用组件**：`Button`, `Card`, `Input`, `Form`, `Upload`。
- **Form 表单校验**：
  - 通过 `rules` 定义校验逻辑。
  - `onFinish` 接收校验通过后的数据。
- **数据回填**：
  ```javascript
  const [form] = Form.useForm();
  form.setFieldsValue({ name: '张三' });
  ```
- **Upload 文件上传**：
  - `action` 指定上传地址。
  - `onChange` 监听上传状态。发送格式通常为 `multipart/form-data`。

## 4. 项目进阶与优化

### 4.1 Token 持久化
- 登录成功后将 Token 存储在 `localStorage` 或 `sessionStorage` 中。
- 初始化 Store 时从本地读取 Token，确保刷新不丢失。

### 4.2 路径别名 (Alias)
- 在 `vite.config.js` 或 `webpack.config.js` 中配置 `@` 指向 `src` 目录。
- 配置 `jsconfig.json` 获得代码提示。

### 4.3 CDN 优化
- 将体积巨大的第三方库（如 React, ReactDOM）通过 CDN 引入。
- 在构建配置中设置 `externals`，减少打包体积。

## 5. 类组件与 React + TS

### 5.1 类组件 (Class Component)
- 继承 `React.Component`。
- 使用 `this.state` 和 `this.setState`。
- 生命周期：`componentDidMount`, `componentWillUnmount` 等。

### 5.2 React + TypeScript (ReactTS)
- **Props 类型**：`interface Props { title: string }`。
- **组件定义**：`const App: React.FC<Props> = ({ title }) => { ... }`。
- **Hooks 类型**：`useState<number>(0)`, `useRef<HTMLDivElement>(null)`。
