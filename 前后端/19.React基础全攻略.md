# 19. React 基础全攻略

React 是用于构建用户界面的 JavaScript 库，核心思想是组件化和声明式渲染。

## 1. 环境搭建与启动

### 1.1 快速创建项目
- **方式**：`npx create-react-app 项目名`
  - `npx` 会临时下载工具并执行，创建完成后不再占用空间。
- **目录结构简化**：
  - `public/`：存放 `index.html`。
  - `src/`：存放核心代码。通常只需保留 `App.js` 和 `index.js`。
- **启动服务**：在项目目录下执行 `npm start`，默认运行在 `3000` 端口。

### 1.2 项目入口解析
- **index.js**：
  ```javascript
  import React from 'react';
  import ReactDOM from 'react-dom/client';
  import App from './App';

  const root = ReactDOM.createRoot(document.getElementById('root'));
  root.render(<App />); // 将根组件渲染到 HTML 的 #root 标签中
  ```

## 2. JSX 语法详解

JSX (JavaScript XML) 允许在 JS 中直接编写 HTML 标签。

### 2.1 模板语法
- **插值**：`{ 数据 }`。支持字符串、变量、函数调用、对象（仅部分场景）。
  - 如果传入的是 HTML 标签列表，React 会自动按序渲染。
- **条件渲染**：
  - `&&`：`{ 条件 && <p>显示我</p> }` (逻辑与)。
  - `? :`：`{ 条件 ? <A /> : <B /> }` (三元运算符)。
- **列表渲染**：
  - 使用 `.map()` 遍历数据。
  - **必须绑定 Key**：`<div key={item.id}></div>`。Key 是唯一标识，用于提高 React 的渲染效率。

### 2.2 事件绑定
- **基本写法**：`onClick={函数名}`。
- **传参写法**：`onClick={() => 函数名(参数)}`。
- **事件对象 `e`**：`onClick={(e) => 函数名(e, 参数)}`。在函数内部通过 `e.target` 获取触发事件的元素。

## 3. 组件系统

### 3.1 定义与调用
- **定义**：定义一个**首字母大写**的函数，返回值为 JSX。
- **调用**：使用 `<组件名 />` 或 `<组件名></组件名>`。

### 3.2 组件通信 (重点)
1. **父传子 (Props)**：
   - 父组件：`<Son msg="hello" />`
   - 子组件：`function Son(props) { return <div>{props.msg}</div> }` (props 是只读的)。
2. **传递子元素 (children)**：
   - 父组件：`<Son><p>我是子元素</p></Son>`
   - 子组件：通过 `props.children` 获取并渲染。
3. **子传父 (状态提升)**：
   - 父组件向子组件传递一个函数（如 `setVar`）。
   - 子组件调用该函数并传入参数，从而修改父组件的状态。
4. **跨多层通信 (Context)**：
   - **顶层**：`const MsgContext = createContext();` 使用 `<MsgContext.Provider value={data}>` 包裹。
   - **底层**：`const data = useContext(MsgContext);` 直接获取。

## 4. Hooks 函数 (核心)

Hooks 只能在组件函数顶层调用，不能放在 `if` 或 `for` 中。

### 4.1 useState (状态管理)
- `const [state, setState] = useState(初始值);`
- **更新规则**：
  - 必须使用 `setState` 修改，不能直接 `state = ...`。
  - **异步性**：`setState` 不会立即生效，而是推入下一次事件循环。React 会收集多个 `setState` 后统一重新渲染（批处理）。
  - **比较机制**：新旧值相等时不触发更新（引用类型需创建新对象/数组）。

### 4.2 useRef (操作 DOM)
- `const myRef = useRef(null);`
- 绑定：`<div ref={myRef}></div>`
- 访问：`myRef.current`。

### 4.3 useEffect (副作用处理)
用于处理数据获取、订阅、定时器等渲染后的操作。
- `useEffect(() => { ... }, [依赖项])`
- **依赖项参数**：
  - 不传：每次渲染后都执行。
  - `[]`：仅在初始渲染（挂载）后执行一次。
  - `[var]`：初始渲染及 `var` 变化时执行。
- **清理函数**：`useEffect` 返回的函数会在组件卸载或下次 Effect 执行前调用。

### 4.4 useMemo & useCallback (性能优化)
- **useMemo**：缓存计算结果。仅当依赖项变化时才重新计算。
- **useCallback**：缓存函数引用。防止父组件重渲染导致子组件接收到新的函数引用而无效重渲染。

### 4.5 useLocation
- 获取当前 URL 信息（需配合 `react-router-dom`）。
- `const location = useLocation(); path = location.pathname;`

---

## 5. 开发原则与样式

- **State 最小化**：能推导出的数据不要设为 State。
- **渲染与副作用分离**：逻辑放在函数体中，副作用（如 API 请求）必须放在 `useEffect` 中。
- **CSS 导入**：`import './xxx.css'`。React 支持直接导入 CSS 文件并应用类名。
