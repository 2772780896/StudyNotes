# Git 版本控制实战

Git 是目前全球最主流的分布式版本控制系统，是程序员协作的基石。

## 1. 基础配置

- **版本查看**：`git -v`
- **配置用户信息**：
  ```bash
  git config --global user.name "YourName"
  git config --global user.email "email@example.com"
  ```
- **查看配置**：`git config --list`

## 2. 核心工作流

1. **初始化**：`git init` (生成 `.git` 目录)
2. **暂存 (Staging)**：`git add <file>` 或 `git add .` (全部暂存)
3. **提交 (Commit)**：`git commit -m "feat: login feature"`
4. **查看状态**：`git status -s` (简短模式)
   - `??`: 未跟踪
   - `A`: 新增到暂存区
   - `M`: 已修改

## 3. 版本回溯与撤销

- **查看日志**：`git log --oneline`
- **工作区回滚**：`git restore <file>` (用暂存区覆盖工作区)
- **重置 (Reset)**：
  - `--soft <commit>`：撤销提交，保留代码在**暂存区**（常用，用于合并提交）。
  - `--mixed <commit>`：默认模式，撤销提交和暂存，代码保留在**工作区**。
  - `--hard <commit>`：**危险！** 彻底清除工作区和暂存区的所有修改。

## 4. 分支管理 (Branching)

- **创建分支**：`git branch <name>`
- **切换分支**：`git checkout <name>` 或 `git switch <name>`
- **创建并切换**：`git checkout -b <name>`
- **合并分支**：`git merge <branch>`
  - 如果有冲突，需手动修改文件后 `git add` 并 `git commit`。
- **删除分支**：`git branch -d <name>`
- **暂存当前修改 (补充)**：`git stash` (当你不想提交但想切换分支时使用)。

## 5. 远程仓库协作

- **绑定远程**：`git remote add origin <url>`
- **推送**：`git push -u origin <branch>`
- **拉取**：
  - `git clone <url>`：克隆整个仓库。
  - `git pull`：拉取并合并。
  - `git pull --rebase`：建议使用，保持提交历史整洁。
- **代理设置**：
  ```bash
  git config --global http.proxy http://127.0.0.1:port
  git config --global --unset http.proxy # 取消代理
  ```

## 6. 忽略文件 (.gitignore)

在项目根目录创建 `.gitignore`，写入需要忽略的文件名或路径（如 `node_modules/`, `.env`）。

---
**复习提示**：日常开发最常用的流程是：`git pull` -> 开发 -> `git add .` -> `git commit` -> `git push`。
