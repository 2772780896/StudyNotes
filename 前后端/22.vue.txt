            这是一个非常关键的问题。在 Vue 的模板（HTML 部分）中， 并不是所有双引号里的内容都会被识别为变量 。Vue 有一套严格的“前缀规则”来区分什么是 普通字符串 ，什么是 变量/表达式 。

为了方便你理解，我们可以把 HTML 属性分为三类：

### 1. 静态属性（普通 HTML 模式）
如果没有特殊前缀，双引号里的内容永远是 普通字符串 。

- 例子 ： <el-form label-position="top">
- 解析 ：Vue 会把 "top" 当作一串普通的字母传给组件，就像原生 HTML 一样。
### 2. 动态绑定（:  或 v-bind 模式） —— 最常用的变量标识
一旦属性名前面加了冒号 : （这是 v-bind 的缩写），双引号里的内容就会被 Vue 的引擎解析为 JavaScript 表达式（变量、函数、数学运算等） 。

- 例子 ： <el-form :model="loginForm"> —— Login.vue:L10
- 解析 ：因为有 : ，Vue 不会去找名为 "loginForm" 的字符串，而是去 <script> 里的 ref 或 reactive 中找名为 loginForm 的 变量 。
### 3. 指令模式 (v-model, v-if, v-for)
以 v- 开头的指令，双引号内部 默认就是变量或表达式 ，不需要加冒号。

- 例子 ： <el-input v-model="loginForm.username"> —— [Login.vue:L13](file:///e:/学习相关/实战项目/基于Vue+Axios的好物推荐 systems/src/views/Login.vue#L13)
- 解析 ： v-model 是 Vue 的内置指令，它规定了引号内必须是一个可读写的 变量路径

    使用路由
    import { useRoute, useRouter } from 'vue-router'
    const route = useRoute()
    const router = useRouter()

    useRoute相当于 React 的 useParams + useLocation 。
    - 它返回的是当前激活路由的状态对象。
    - 包含了当前 URL 的解析信息，如 params 、 query 、 path 、 fullPath 等。
    - 原理 ：它是一个 Proxy 响应式对象 。这意味着如果路由参数发生变化（比如从 /detail/1 跳转到 /detail/2 ）， route.params 会自动更新，触发 Vue 的响应式系统。
    通过route.params.id得到设定时传入的参数

    useRouter() ：相当于 React 的 useNavigate 。
    - 它返回的是全局路由实例（Router Instance）。
    - 用于执行编程式导航，比如 router.push() 、 router.replace() 、 router.back() 。
    // router.push({
    //   path: '/',
    //   query: { q: searchText.value }  可选，在跳转目标地址后加上 ?q=searchText.value
    // })
    - 原理 ：它提供了操作历史记录栈（History Stack）的方法。

    路由守卫
    // 处理登录拦截
    router.beforeEach((to, from, next) => {
        // - to ：即将要进入的目标路由对象。
        // - from ：当前导航正要离开的路由对象。
        // - next ：这是一个 必须调用 的函数，用来决定导航的下一步。
        
        const userStore = useUserStore()
  
        // 动态设置页面标题
        if (to.meta.title) {
            document.title = `${to.meta.title} - 好物推荐系统`
        }

        // 拦截逻辑
        if (to.meta.requiresAuth && !userStore.isLoggedIn) {
        // 如果页面需要登录但未登录，跳转到登录页
            next('/login')
            // - next() ：放行。
            // - next('/login') ：中断当前导航，强制跳转到登录页。
        } else if (to.path === '/login' && userStore.isLoggedIn) {
        // 如果已登录还想去登录页，直接送回首页
            next('/')
        } else {
            next()
        }
    })


axios
    封装
    import axios from 'axios'
    const service = axios.create({
        baseURL: 'http://localhost:3000', // 1. 基础地址：以后发请求只需要写后缀，比如 /goods
        timeout: 5000                     // 2. 超时：如果 5 秒没响应，就报错
    })

    // 响应拦截器：数据回来后的“安检口”
    service.interceptors.response.use(
        response => {
        // Axios 默认会把数据包在 response.data 里
        // 我们在这里直接返回 response.data，这样组件里拿到的就是纯净的数据库数据
            return response.data
        },
        error => {
            // 如果后端崩了或断网，这里会统一弹出一个红色的错误提示
            ElMessage.error(error.message || '网络请求错误')
            return Promise.reject(error)
        }
    )
    export default service

        并发等待
        const [allGoods, allActions] = await Promise.all([
            request.get('/goods'),
            request.get('/actions')
        ])
        1. Promise.all([ ... ]) ：
            它接收一个 数组 ，数组里放的是你想同时进行的异步任务（Promise）。
        2. await ：
            它会“等”数组里 所有的 任务都完成了，才会继续往下走。如果其中任何一个失败了，整个 Promise.all 就会报错（抛出异常）。
        3. const [allGoods, allActions] = ... (解构赋值) ：
            这是 ES6 的数组解构语法。
            - Promise.all 返回的结果也是一个数组，顺序和你传入的顺序一致： [task1的结果, task2的结果] 。
            - 通过 [allGoods, allActions] ，我们直接把第一个结果存入 allGoods 变量，第二个结果存入 allActions 变量。

        如果不用Promise.all，会变成：
            const allGoods = await request.get('/goods');   
            const allActions = await request.get('/actions'); 
        第二个request需要等待第一个await结束才能开始执行

pinia
    创建状态
    import { defineStore } from 'pinia'
    import { ref } from 'vue'

    // defineStore用于创建一个状态，第一个参数为状态名，第二个参数为用于设置该状态内容的函数
    export const useUserStore = defineStore('user', () => {
        
        // 用 ref() 定义的变量就是全局状态
        const currentUserId = ref('1001')

        // 定义的普通函数就是修改状态的方法
        const setCurrentUser = (id) => {
            currentUserId.value = id
        }

        // 用 computed() 定义的变量就是派生状态

        // 暴露给外部使用
        return {
            currentUserId,
            userList,
            setCurrentUser
        }
    }    

    使用状态
    import { useUserStore } from '../store/user'

    // 拿到状态，当状态没有时会创建一个实例，当多次在其他组件调用时，Pinia 会直接返回之前创建好的那个实例
    const userStore = useUserStore()

    // 读取状态
    const userId = userStore.currentUserId

    // 调用状态的方法来修改状态
    userStore.setCurrentUser('1002')
    // 因为 currentUserId 是一个 ref ，所以当一个组件修改了它， 所有 监听这个 Store 的组件都会因为 Vue 的响应式系统而自动更新
