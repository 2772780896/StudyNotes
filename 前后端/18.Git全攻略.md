# 18. Git 全攻略

Git 是目前最流行的分布式版本控制系统，能够高效处理从小型到大型的项目版本管理。

## 1. 安装与初始配置

### 1.1 基本设置
安装 Git 后，在 Git Bash 中执行：
- **查看版本**：`git -v`
- **查看配置列表**：`git config --list`
- **配置用户信息** (全局)：
  - `git config --global user.name "你的用户名"`
  - `git config --global user.email "你的邮箱"`

---

## 2. 基础操作：从本地到仓库

### 2.1 创建仓库
- **初始化**：`git init`
  - 在当前目录下创建一个 `.git` 隐藏文件夹，该目录下的文件即进入工作区。

### 2.2 提交到暂存区 (Staging Area)
- **添加文件**：`git add 文件路径` (使用 `git add .` 添加所有改动)。
- **查看暂存区文件**：`git ls-files`
- **从暂存区移除**：`git rm --cached 文件路径`

### 2.3 提交到版本库 (Repository)
- **提交**：`git commit -m "提交注释"`
  - 将暂存区内容永久保存到版本库，生成一个 7 位 16 进制的版本号。
- **查看提交记录**：`git log --oneline`

### 2.4 缓存与清理
- **删除暂存区及索引**：`git rm --cached -r 文件路径`
  - 场景：如果某个子目录之前有 `.git` 仓库（嵌套仓库），即使删除了子目录的 `.git`，父仓库可能仍会忽略它。此时需要清除该子目录的缓存后重新 add。

---

## 3. 文件状态与回溯

### 3.1 文件标识解析
- `U` (Untracked)：未被追踪的文件。
- `A` (Added)：第一次被加入暂存区的文件。
- `M` (Modified)：已修改的文件。
- `''` (无标识)：工作区、暂存区、版本库内容一致。
- **快速查看状态**：`git status -s`
  - 返回格式：`标识符1标识符2 文件名`
  - `标识符1` 代表暂存区状态，`标识符2` 代表工作区状态。

### 3.2 数据回溯 (后悔药)
- **回溯工作区**：`git restore 文件路径` (用暂存区覆盖工作区)。
- **版本回退 (Reset)**：
  - `git reset --soft 版本号`：保留工作区和暂存区的修改，仅撤销 commit。
  - `git reset --mixed 版本号` (默认)：保留工作区修改，清空暂存区。
  - `git reset --hard 版本号`：**危险！** 清空工作区和暂存区所有未提交的修改，彻底回到指定版本。

---

## 4. 分支管理 (Branching)

Git 默认主分支名为 `master`。`HEAD` 是一个指针，指向当前正在使用的分支。

- **查看分支**：`git branch`
- **创建分支**：`git branch 分支名`
- **切换分支**：`git checkout 分支名` (或 `git switch 分支名`)
- **合并分支**：`git merge 分支名`
  - 将目标分支的内容合并到当前分支。
  - **冲突处理**：如果两个分支修改了同一文件的同一行，合并会暂停。你需要手动打开文件，选择保留的代码，然后重新 `add` 和 `commit`。
- **删除分支**：`git branch -d 分支名`

---

## 5. 远程仓库协作

### 5.1 绑定与查看
- **绑定**：`git remote add 别名 地址` (常用别名为 `origin`)。
- **解绑**：`git remote remove 别名`
- **查看远程**：`git remote -v`

### 5.2 上传与拉取
- **克隆**：`git clone 仓库地址`
  - 自动创建文件夹、初始化 git、添加远程绑定并拉取代码。
- **推送**：`git push -u 别名 本地分支:远程分支`
  - 第一次推送建议加 `-u` 以建立追踪关系，后续只需 `git push`。
- **拉取合并**：`git pull 别名 远程分支:本地分支`
- **强制拉取**：`git pull --rebase 别名 远程分支`
  - 当远程有无关提交（如在网页上直接修改了 README）导致无法直接 push 时使用。

### 5.3 代理配置 (解决访问慢)
- **设置代理**：
  - `git config --global http.proxy http://服务器地址:端口`
  - `git config --global https.proxy https://服务器地址:端口`
- **查看/取消代理**：
  - 查看：`git config --global --get https.proxy`
  - 取消：`git config --global --unset https.proxy`

---

## 6. 其他功能

- **.gitignore**：在该文件中列出的文件或目录将不会被提交到仓库（如 `node_modules/`, `.env`）。
- **GitHub Page**：通过远程仓库设置，可以将仓库中的 HTML 文件直接解析为在线网页。
