# 17. TypeScript 全攻略

TypeScript (TS) 是 JavaScript 的超集，为 JS 增加了静态类型检查。它能帮助我们在编写代码时就发现错误，而不是等到运行时。

## 1. 环境搭建与运行

### 1.1 安装与编译
- **安装 TS**：`npm i -g typescript`
- **编译 TS 文件**：`tsc 文件名.ts`
  - 这会在同级目录下生成一个同名的 `.js` 文件。之后你可以使用 `node 文件名.js` 来运行。
- **ts-node (推荐)**：简化运行流程。
  - **安装**：`npm i -g ts-node`
  - **运行**：`ts-node 文件名.ts`（它会在内存中将 TS 编译为 JS 并执行，不会生成物理 `.js` 文件）。

---

## 2. 类型系统基础

### 2.1 类型注解
- **语法**：`let 变量名: 类型 = 值`
- **特性**：一旦指定类型，赋值不同类型的值会报错。

### 2.2 基本类型
- `number`, `string`, `boolean`, `null`, `undefined`
- **联合类型**：`(类型1 | 类型2)`，表示变量可以是其中任何一种类型。
- **自定义类型 (Type Alias)**：
  - `type 驼峰式类型名 = 类型`
  - 示例：`type MyType = string | number;`

### 2.3 字面量类型
- 变量的值必须与指定的字面量完全一致。
- 示例：`let str: 'hello' = 'hello';`（常用于配置项，如 `type Gender = 'man' | 'woman'`）。

### 2.4 Any 类型 (慎用)
- `any`：不对变量进行任何类型约束，相当于回到了普通的 JS。

---

## 3. 复杂类型

### 3.1 数组类型
- **写法 1**：`类型[]`（推荐，如 `number[]`）。
- **写法 2**：`Array<类型>`（不常用）。

### 3.2 元组类型 (Tuple)
- 指定数组的元素个数和每一项的具体类型。
- 示例：`let position: [number, string] = [1, 'center'];`

### 3.3 函数类型
- **直接注解**：
  - `function add(a: number, b: number): number { return a + b; }`
  - `const add = (a: number, b: number): number => { return a + b; }`
- **单独注解函数变量**：
  - `const add: (a: number, b: number) => number = (a, b) => a + b;`
- **特有类型 `void`**：当函数没有返回值时使用。
- **可选参数**：在参数名后加 `?`。
  - 注意：可选参数必须放在必选参数之后。
  - `function greet(name: string, age?: number) { ... }`

---

## 4. 对象与接口 (Interface)

### 4.1 对象注解
- `let user: { name: string; age: number; sayHi(): void } = { ... }`
- **可选键**：`键名?: 类型`。

### 4.2 接口 (Interface)
- **定义**：`interface IUser { name: string; age: number }`
- **用途**：专门用于定义对象结构。相比 `type`，接口更专注于对象。
- **接口继承**：
  ```typescript
  interface Point2D { x: number; y: number }
  interface Point3D extends Point2D { z: number } // 获得 x, y, z
  ```
- **交叉类型 (Intersection)**：
  - `type Point3D = Point2D & { z: number }`
  - 效果类似继承，通过 `&` 获得多个类型的属性。
- **冲突处理**：
  - **继承**：如果子接口与父接口有同名但类型不同的属性，会报错。
  - **交叉**：如果有同名属性，该属性类型会变为 `类型1 & 类型2`（如果冲突，则可能变为 `never`）。

---

## 5. 高级特性

### 5.1 索引签名
- **对象索引**：不知道具体键名，只知道键和值的类型。
  - `interface MyObj { [key: string]: number }`（键是字符串，值是数字）。
- **数组索引**：
  - `interface MyArr { [key: number]: string }`（键是数字，本质上就是数组）。

### 5.2 枚举 (Enum)
- **语法**：`enum Direction { Up, Down, Left, Right }`
- **值**：默认从 0 开始自增（0, 1, 2, 3）。
- **手动赋值**：`enum Color { Red = 10, Green = 20 }`
- **特性**：枚举会被编译成 JS 对象，存在运行时开销。

### 5.3 类 (Class)
- **基本使用**：`class Person { name: string; constructor(n: string) { this.name = n; } }`
- **可见性修饰符**：
  - `public` (默认)：任何地方可访问。
  - `protected`：仅在当前类和子类中可访问。
  - `private`：仅在当前类内部可访问。
- **实现接口 (implements)**：类必须实现接口定义的所有属性和方法。
  - `class Student implements IUser { ... }`

---

## 6. 类型工具与操作

### 6.1 类型推论、断言与 typeof
- **类型推论**：TS 会根据初始值自动推断类型。
- **类型断言 (Type Assertion)**：手动告诉编译器变量的类型。
  - 语法：`变量 as 类型`。
- **typeof**：获取变量或对象的类型。
  - `let p1 = { x: 1, y: 2 }; let p2: typeof p1;`

### 6.2 类型兼容性
- TS 采用 **结构化类型系统**。
- **核心逻辑**：如果类型 A 包含了类型 B 的所有必需属性，那么 A 的实例可以赋值给 B 类型的变量（“多给少”是兼容的）。

### 6.3 泛型 (Generics) - 复用之魂
泛型是指在定义时不指定具体类型，而在使用时才传入。
- **泛型函数**：`function id<T>(arg: T): T { return arg; }`
- **泛型约束**：限制传入的泛型必须满足某种条件。
  - `function log<T extends { length: number }>(arg: T) { console.log(arg.length); }`
- **泛型接口**：`interface Box<T> { value: T }`
- **泛型类**：`class Container<T> { ... }`

### 6.4 泛型工具类
- `Partial<T>`：将所有属性变为可选。
- `Readonly<T>`：将所有属性变为只读。
- `Pick<T, K>`：从 T 中挑选部分属性 K。
- `Record<K, T>`：创建一个键为 K，值为 T 的对象类型。

---

## 7. 映射类型 (高级)

- **in 运算符**：遍历联合类型。
  - `type Keys = 'a' | 'b'; type Obj = { [K in Keys]: number };` // { a: number, b: number }
- **keyof**：获取对象类型的所有键，返回联合类型。
  - `type UserKeys = keyof { name: string; age: number };` // 'name' | 'age'
- **索引查询**：
  - `type User = { name: string; age: number }; type NameType = User['name'];` // string

---

## 8. 类型声明文件 (.d.ts)

- **作用**：仅包含类型信息，不包含执行代码。用于为 JS 提供类型支持。
- **获取声明文件**：
  - 包自带：查看 `package.json` 中的 `typings` 或 `types` 字段。
  - 外部安装：`npm i -D @types/包名` (如 `@types/lodash`)。
- **自定义声明**：使用 `declare` 关键字为 JS 中的变量、函数等提供声明。
