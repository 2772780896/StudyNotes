# 21. Mockjs 全攻略

Mockjs 是一款前端模拟数据生成器，允许前端在后端接口未就绪时独立开发，并支持生成各种类型的随机数据。

## 1. 安装与初始化

- **安装**：`npm i mockjs`
- **引入**：在 React/Vue 的入口文件（如 `index.js` 或 `main.js`）中执行 `import './mock'`。
- **配置目录**：通常在 `src/mock/index.js` 中编写拦截逻辑。

---

## 2. 核心语法：Mock.mock()

### 2.1 基本用法
`Mock.mock(url, type, template)`
- **url**：拦截的请求路径（支持字符串或正则）。
- **type**：请求类型（get, post, put, delete）。
- **template**：响应数据的模板对象。

### 2.2 数据模板定义 (语法规范)
- **数组**：`"list|1-10": []` (随机生成 1 到 10 个元素的列表)。
- **字符串**：`"string|1-10": ""` (随机生成 1 到 10 个重复字符)。
- **数字**：
  - `"number|1-100": 1` (1 到 100 的整数)。
  - `"float|1-100.1-10": 1` (整数 1-100，小数 1-10 位)。
- **布尔值**：
  - `"bool|1": true` (50% 概率)。
  - `"bool|2-5": false` (概率为 5/(2+5))。
- **对象**：`"obj|2-4": { a: 1, b: 2, c: 3, d: 4 }` (随机抽取 2 到 4 个属性)。

---

## 3. Mock.Random (占位符)

Random 工具类提供了极其丰富的随机数据生成方法。可以使用 `Random.xxx()` 或在模板中使用 `'@xxx'`。

### 3.1 常用占位符
- **文本**：
  - `@cword(min, max)`：随机中文字。
  - `@ctitle(min, max)`：随机中文标题。
  - `@cparagraph()`：随机中文段落。
- **个人信息**：
  - `@cname()`：随机中文名。
  - `@email('163.com')`：指定域名的邮箱。
- **地理/网络**：
  - `@ip()`：随机 IP 地址。
  - `@color()`：随机 16 进制颜色。
- **日期时间**：
  - `@datetime('yyyy-MM-dd HH:mm:ss')`：格式化日期。
  - `@time()`：随机时间。
- **图片**：
  - `Random.image(size, background, foreground, format, text)`：生成占位图 URL。
  - `Random.dataImage(size, text)`：生成 Base64 编码图片。

## 4. 复杂请求拦截与参数获取

当需要根据请求参数返回不同数据时，可以使用函数形式。

### 4.1 获取 GET 请求参数
由于 GET 参数在 URL 中，需要使用正则表达式匹配并手动解析。
```javascript
Mock.mock(/\/api\/list(\?.*)?$/, 'get', (options) => {
  const url = new URL(options.url, 'http://localhost');
  const param = url.searchParams.get('id'); // 获取参数
  return Mock.mock({
    code: 200,
    data: { id: param, name: '@cname' }
  });
});
```

### 4.2 获取 POST 请求参数
POST 参数通常在请求体中。
```javascript
Mock.mock('/api/login', 'post', (options) => {
  const body = JSON.parse(options.body); // 解析请求体
  return {
    code: 200,
    message: '成功',
    _headers: { // 模拟设置响应头
      'Set-Cookie': 'session_id=123; Path=/'
    }
  };
});
```

### 4.3 获取请求头 (Header)
- 通过 `options.xhr.requestHeaders` 获取前端发送的请求头信息。

---

## 5. 注意事项

- **网络面板不可见**：Mockjs 是在 JS 层面拦截请求，因此浏览器的 Network 面板中**看不到**这些请求。
- **生产环境禁用**：发布项目前，务必移除 mock 导入，或通过环境变量判断是否启用。
