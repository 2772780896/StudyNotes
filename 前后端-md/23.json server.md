## db.json 本质上是一个标准的 JSON 对象，其**顶级键（Top-level Keys）**即为 API 的资源路径。

## - 基本格式要求 ：
## - 必须是合法的 JSON 格式（键名必须加双引号）。
## - 每个资源集合必须是一个 数组 。
## - 每个对象必须包含一个唯一的 id 字段（ json-server 默认以 id 作为主键）。

## - 项目示例 db.json ：
## {
### "goods": [ { "id": "1", "name": "耳机", "categoryId": "101" } ],
### "categories": [ { "id": "101", "name": "数码" } ],
### "actions": []
## }

## 启动
```
npx json-server --watch db.json --port 3000
```

### - --watch ：监听文件变化，一旦 db.json 改变，接口数据实时更新。
### - --port ：指定端口（默认 3000）。

## RESTful关键字与对应数据
## GET /goods 获取全部商品列表 数组 [...]
## GET /goods/1 获取 ID 为 1 的商品详情 对象 {...}
## POST /goods 创建一个新商品 新对象 （包含自动生成的 ID）
## PUT /goods/1 替换 ID 为 1 的商品全部信息 更新后的 完整对象
## PATCH /goods/1 局部更新 ID 为 1 的商品部分字段 更新后的 完整对象
```
DELETE /goods/1 删除 ID 为 1 的商品 空对象 {}

```

## 请求传参
## - 过滤 ： /goods?`category=数码`
## - 分页 ： /goods?`_page=1`&`_limit=10`
## - 排序 ： /goods?`_sort=price`&`_order=asc` （ asc 升序， desc 降序）
## - 搜索 ： /goods?`q=索尼` （在所有字段中模糊匹配“索尼”）
## - 关系提取 ：
### - /goods?`_expand=category` ：外键关联，在商品数据中加入分类的详细信息。将被关联的数据作为子项插入获取到的数据中
### - /categories/101?`_embed=goods` ：获取分类 101 的同时，带上属于该分类的所有商品。
