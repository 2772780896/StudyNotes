# JSON Server

## 安装

```bash
npm install -D json-server
```

## 数据文件 `db.json`

本质为一个 JSON 对象，顶级键即为 API 资源路径。

```json
{
    "goods": [
        { "id": "1", "name": "耳机", "categoryId": "101" }
    ],
    "categories": [
        { "id": "101", "name": "数码" }
    ],
    "actions": []
}
```

**格式要求：**
- 合法 JSON（键名必须双引号）
- 每个资源集合必须是数组
- 每个对象必须含唯一 `id` 字段（默认主键）

## 启动

```bash
npx json-server --watch db.json --port 3000
```

| 参数 | 说明 |
|------|------|
| `--watch` | 监听文件变化，接口数据实时更新 |
| `--port` | 指定端口（默认 3000） |

## RESTful API

| 方法 | 路径 | 说明 | 返回值 |
|------|------|------|------|
| `GET` | `/goods` | 获取全部 | 数组 `[...]` |
| `GET` | `/goods/1` | 获取 ID=1 详情 | 对象 `{...}` |
| `POST` | `/goods` | 创建 | 新对象（含自动生成 ID） |
| `PUT` | `/goods/1` | 替换全部字段 | 更新后完整对象 |
| `PATCH` | `/goods/1` | 局部更新部分字段 | 更新后完整对象 |
| `DELETE` | `/goods/1` | 删除 | 空对象 `{}` |

## 查询参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `?key=value` | 过滤 | `/goods?categoryId=101` |
| `?_page=1&_limit=10` | 分页 | 第 1 页，每页 10 条 |
| `?_sort=price&_order=asc` | 排序（`asc` 升序/`desc` 降序） | |
| `?q=关键词` | 全字段模糊搜索 | `/goods?q=索尼` |
| `?_expand=category` | 外键关联（将关联数据作为子项插入） | |
| `?_embed=goods` | 反向关联（获取当前资源时带上子资源） | `/categories/101?_embed=goods` |
