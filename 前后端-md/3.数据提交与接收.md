## 数据提交

数据提交有 GET 和 POST 两种方式。

| 方式 | 特点 |
|------|------|
| GET | 提交的数据在 URL 中以 `?name1=value1&name2=value2` 形式可见 |
| POST | 提交的数据不在 URL 中显示 |

### form 表单

```html
<form method="get|post" action="提交地址">
    <input type="submit" />
</form>
```

- 只提交有 `name` 属性的标签，提交的是 `name` 和 `value` 的值
- 也可通过 JavaScript 调用 `表单.submit()` 提交
- `method` 指定提交方式，`action` 指定目标地址

## 数据接收

通过 `request.method` 判断请求方式：

```python
from flask import request

@app.route('/show/info', methods=['GET', 'POST'])
def index():
    if request.method == 'GET':
        data = request.args      # GET 提交的数据
    else:
        data = request.form      # POST 提交的数据
```

- `request.args` 和 `request.form` 的类型是 `ImmutableMultiDict`，类似于字典但允许多个相同键存在，且不可变
- `methods` 参数指定允许的请求方式，默认为 `['GET']`，一个路由可同时支持多种方式

## localStorage 与 Cookie

### Cookie

浏览器响应头中包含 `Set-Cookie`，浏览器自动设置：

```
Set-Cookie: JSESSIONID=abcde12345; Path=/; HttpOnly; SameSite=Lax
```

后续请求自动携带：

```
Cookie: JSESSIONID=abcde12345
```

可通过 `document.cookie` 读写，但设置了 `HttpOnly` 的 Cookie 不可被 JavaScript 访问。

### localStorage

存储在浏览器本地，无过期时间：

```javascript
localStorage.setItem(key, value)
localStorage.getItem(key)
localStorage.removeItem(key)
```

存储 token 时，可在请求头中发送：

```
Authorization: Bearer <token>
```

## RESTful API

- **路径** `/api/videos/{vid}` — 标识唯一资源
- **查询参数** `?key=value` — 用于过滤、排序、分页
