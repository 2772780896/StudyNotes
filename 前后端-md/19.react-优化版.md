## 1. 开发环境搭建：CRA vs Vite

### 1.1 经典方案：Create React App (CRA)
- **命令**: `npx create-react-app react项目名`
- **原理 (npx)**: `npx` 是 npm 5.2+ 引入的工具。它会下载 `create-react-app` 到临时目录，运行完后自动删除，保证你用的永远是最新版，且不占用全局空间。
- **运行**: 
  - `cd react项目名`
  - `npm start` (启动 3000 端口服务)
- **构建原理**: 基于 **Webpack**。Webpack 是“全量打包”模式，它必须先扫描并打包你项目里的所有模块，才能启动开发服务器。因此项目越大，启动越慢。

### 1.2 现代方案：Vite (补充建议)
- **命令**: `npm create vite@latest my-app -- --template react`
- **启动**: `npm run dev`
- **Vite 为什么快？ (核心原理)**:
  - **开发环境**: 利用浏览器原生的 **ES Modules (ESM)**。浏览器在解析到 `import` 时会向服务器发请求，Vite 只在此时进行按需转换。
  - **编译速度**: 使用 Go 语言编写的 **Esbuild** 进行预构建，比 JS 编写的 Webpack 快 10-100 倍。

## 2. 项目结构与入口流程

### 2.1 核心文件夹
- **public/**: 存放静态资源。
  - `index.html`: 整个应用的唯一 HTML 页面，内部有一个 `<div id="root"></div>`。
- **src/**: 存放源码。
  - `index.js` (或 `main.jsx`): 项目入口。
  - `App.js`: 根组件。

### 2.2 入口文件运行流程 (index.js)
```javascript
import React from 'react'; 
import ReactDOM from 'react-dom/client'; 
import App from './App';

// 1. 获取 HTML 中的 root 节点
const root = ReactDOM.createRoot(document.getElementById('root'));
// 2. 将根组件 App 渲染到该节点中
root.render(<App />);
```
- **原理**: React 通过 `ReactDOM` 建立起“虚拟 DOM”与“真实 DOM”的桥梁。`createRoot` 开启了 React 18 的并发模式基础。

## 3. JSX 语法全解：UI 的本质

JSX (JavaScript XML) 是在 JS 中编写 HTML 标签的语法扩展。

### 3.1 JSX 的转换原理 (补充知识)
浏览器不认识 JSX。在构建时，Babel 会将 JSX 转换为原生的 JS 函数调用：
- **JSX**: `<div className="hi">Hello</div>`
- **JS**: `React.createElement('div', { className: 'hi' }, 'Hello')`
这解释了为什么 JSX 中只能有一个根标签（因为函数只能返回一个值）。

### 3.2 核心模板语法
在 JSX 中，使用 `{}` 来包裹 JavaScript 表达式。

#### **(1) 基础插值**
```jsx
const name = 'React';
const obj = { age: 18 };
function getName() { return 'Jack'; }

<div>
  {name}               {/* 变量 */}
  {obj.age}            {/* 对象属性 */}
  {getName()}          {/* 函数调用 */}
  { [1, 2, 3] }        {/* 数组：React 会自动展开并按顺序渲染 */}
</div>
```

#### **(2) 条件渲染**
- **逻辑与 (&&)**: 当条件为真时，渲染后面的数据。
  `{ condition && <div>显示内容</div> }`
- **三元运算符 (? :)**: 相当于 `if-else`。
  `{ isLogin ? <span>已登录</span> : <span>请登录</span> }`

#### **(3) 列表渲染 (.map)**
当需要将对象/数组批量渲染为标签时，必须使用 `.map()`。
```jsx
const list = [
  { id: 1, name: 'Vue' },
  { id: 2, name: 'React' }
];

<ul>
  {list.map(item => (
    <li key={item.id}>{item.name}</li>
  ))}
</ul>
```
- **Key 的深度原理**: React 使用虚拟 DOM。当列表变化时，React 通过 `key` 来快速识别哪些项被改变、添加或删除了。如果没有唯一的 `key`（或使用 index 作为 key），在列表发生增删/排序时，会导致 DOM 节点的错误复用，产生严重的性能问题或 UI 渲染 Bug。

**(4) ...展开属性语法**
JSX 的属性区里的 `{ }` 里，`...` 不是对象的 `{...}`，是 属性展开：
```jsx
// 普通 JS： { } 可以是对象字面量 
const copy = { ...props }       // 新对象 = { video: item } 

// JSX 属性区： { } 是属性容器，... 是"把对象里的键值对拆成 prop" 
<Card {...props} />             // 等价于 <Card video={item} />

// JSX 内容区； { } 依旧是普通的展开运算符
<Card>{...props}</Card>         // 内容位置 → ❌ 就是普通对象展开
```

### 3.3 事件绑定与传参
React 事件采用小驼峰命名法（如 `onClick`）。

- **基础绑定**:
  `onClick={handleClick}` (注意不能加括号，加了括号会立即执行)
- **传递自定义参数**:
  `onClick={() => handleClick(123)}` (利用匿名箭头函数包裹)
- **获取事件对象 (e)**:
  - **默认情况**: `const handleClick = (e) => { console.log(e) }`
  - **带参数情况**: `onClick={(e) => handleClick(e, 123)}`
  - **原理**: 此时 `e` 也需要在函数定义时被接收。`e` 是 React 封装的“合成事件 (SyntheticEvent)”，旨在解决不同浏览器的兼容性问题。

## 5. 组件系统：React 的基石

组件允许我们将 UI 拆分为独立、可复用的代码片段。

### 5.1 组件定义与调用
- **定义**: 定义一个首字母大写的函数。函数的返回值为 JSX。
- **调用**: 可以通过 `<组件名 />` 或 `<组件名></组件名>` 的方式调用。
- **原理**: 调用组件本质上是执行该函数并获取其返回的虚拟 DOM 结构。

## 6. 组件通信：数据流动教程

React 遵循**单向数据流**原则，数据主要通过 Props 从上往下传递。

### 6.1 父传子 (Props)
- **语法**:
  ```jsx
  // 父组件
  function Parent() {
    const name = 'React';
    return <Son name={name} age={18} />;
  }

  // 子组件接收
  function Son(props) {
    // props 是一个对象，包含了父组件传递的所有数据
    return <div>{props.name} - {props.age}</div>;
  }

  // 推荐：解构赋值写法
  function Son({ name, age }) {
    return <div>{name} - {age}</div>;
  }
  ```
- **核心特性**:
  1. **只读性**: 子组件绝对不能修改 `props` 中的数据。
  2. **传递内容**: 可以传递数字、字符串、布尔值、数组、对象、函数，甚至是 JSX（插槽）。

### 6.2 插槽通信 (children 属性)
- **场景**: 当父组件想把一段 JSX 结构传递给子组件并在特定位置渲染时。
- **语法**:
  ```jsx
  // 父组件
  <Son>
    <div>我是嵌套的内容</div>
  </Son>

  // 子组件
  function Son(props) {
    return (
      <div className="container">
        {props.children} {/* 渲染父组件嵌套的内容 */}
      </div>
    );
  }
  ```

### 6.3 子传父 (回调函数形式)
- **原理**: React 不允许直接“向上”传值。必须由父组件传递一个**修改状态的函数**给子组件，子组件通过调用该函数并传参来实现数据回传。
- **语法流程**:
  1. 父组件准备一个函数。
  2. 将函数通过 props 传给子组件。
  3. 子组件通过 `props.函数名(数据)` 调用。
  ```jsx
  function Parent() {
    const [msg, setMsg] = useState('');
    const getSonMsg = (data) => setMsg(data); // 准备接收函数

    return (
      <div>
        子组件说：{msg}
        <Son onGetMsg={getSonMsg} />
      </div>
    );
  }

  function Son({ onGetMsg }) {
    return <button onClick={() => onGetMsg('妈妈我饿了')}>回传数据</button>;
  }
  ```

### 6.4 跨多层级通信 (Context)
- **场景**: 避免“属性钻取”（Prop Drilling），即数据不需要经过中间组件层层传递。
- **语法流程**:
  1. **创建上下文**: `const MsgContext = createContext()`
  2. **提供数据 (Provider)**: 在顶层组件用 `<MsgContext.Provider value={数据}>` 包裹。
  3. **消费数据 (useContext)**: 在底层组件通过 `useContext(MsgContext)` 获取数据。
  ```jsx
  // 1. 创建
  const MsgContext = createContext();

  // 2. 顶层
  function Top() {
    return (
      <MsgContext.Provider value="我是爷爷的数据">
        <Bottom />
      </MsgContext.Provider>
    );
  }

  // 3. 底层
  function Bottom() {
    const msg = useContext(MsgContext);
    return <div>{msg}</div>;
  }
  ```

## 7. 样式与资源导入
- **CSS 导入**: `import './xxx.css'`。
- **原理**: 构建工具（Vite/Webpack）会识别 CSS 导入，将其提取并自动插入到 HTML 的 `<head>` 标签中。
- **ClassName**: 在 JSX 中，HTML 的 `class` 属性必须写成 `className`，因为 `class` 在 JS 中是保留字（用于定义类）。

## 8. Hooks 核心函数：状态与副作用

Hooks 是 React 16.8 引入的特性，允许在函数组件中使用状态和其他 React 特性。
**调用规则**: 只能在组件函数顶层调用，不能在循环、条件或嵌套函数中调用。

### 8.1 useState (状态管理)
- **语法**: `const [state, setState] = useState(initialValue);`
  - `state`: 当前状态值。
  - `setState`: 修改状态的函数。
- **底层原理**:
  1. **重新渲染 (Re-render)**: 当调用 `setState` 时，React 会重新执行整个组件函数。
  2. **状态持久化**: 尽管组件函数重新执行，当重新运行到`useState`时，不会使用初始的状态变量，而是拿到state的新值，将其作为新的状态变量，。
  3. **异步更新与批处理 (Batching)**: 
     - 调用 `setState` 时，组件并不会立刻重新开始渲染，而是将其放入队列，并且使用本次循环中useState获取到的值来继续运行剩下的代码。
     - 当本次所有代码执行完后，React 会将同一事件循环内的多次 `setState` 合并为一次渲染，并将值设为useState的值，以提升性能。
- **语法示例 (双向绑定)**:
  ```jsx
  const [value, setValue] = useState('');
  // 绑定 input：通过 e.target.value 获取输入并更新状态
  <input value={value} onChange={(e) => setValue(e.target.value)} />
  ```

### 8.2 useRef (引用与持久化)
- **原理**: 返回一个带有 `.current` 属性的对象。该对象在组件的整个生命周期内保持不变。
- **核心场景**:
  1. **操作 DOM**: 替代原生的 `document.getElementById`。
  2. **存储变量**: 存储那些不需要触发重新渲染的数据（如定时器 ID）。
- **语法示例**:
  ```jsx
  const inputRef = useRef(null);
  const handleClick = () => {
    console.log(inputRef.current); // 获取真实的 DOM 节点
  };
  return <div ref={inputRef}>内容</div>;
  ```

### 8.3 useEffect (副作用处理)
- **原理**: 专门用于处理“副作用”（如 API 请求、手动操作 DOM、订阅、定时器）。它将外部系统与 React 的状态同步。
- **执行流程**: 
  1. 浏览器完成渲染并绘制。
  2. 执行 `useEffect` 中的代码。
- **语法结构**:
  ```jsx
  useEffect(() => {
    // 副作用逻辑
    const timer = setInterval(() => {}, 1000);

    // 返回清理函数 (Cleanup)
    return () => clearInterval(timer); 
  }, [依赖项]);
  ```
- **依赖项详解**:
  - `无`: 初始渲染 + 每次更新后都执行。
  - `[]`: 仅在组件挂载 (Mount) 后执行一次。
  - `[data]`: 初始渲染 + `data` 变化时才执行。
- **清理函数 (Cleanup)**: 在组件卸载或下一次 Effect 执行前调用，用于清除定时器、取消订阅等。

### 8.4 useMemo 与 useCallback (性能优化)
- **useMemo**: 缓存**计算结果**。只有依赖项变化时才重新计算。
  `const result = useMemo(() => computeExpensiveValue(a, b), [a, b]);`
- **useCallback**: 缓存**函数引用**。
  - **原因**: 每次组件重绘都会重新定义函数，导致函数引用地址变化。如果该函数作为 props 传给子组件，会导致子组件误以为数据变化而产生多余的重绘。
  `const fn = useCallback(() => { ... }, [deps]);`

### 8.5 自定义 Hook
- **本质**: 抽离业务逻辑的 JS 函数。
- **命名规范**: 必须以 `use` 开头。
- **语法示例**:
  ```jsx
  function useWindowWidth() {
    const [width, setWidth] = useState(window.innerWidth);
    useEffect(() => {
      const handleResize = () => setWidth(window.innerWidth);
      window.addEventListener('resize', handleResize);
      return () => window.removeEventListener('resize', handleResize);
    }, []);
    return width;
  }
  ```

## 9. 最佳实践与原则
1. **最小化 State**: 能通过其他状态计算出来的数据不要存为 State（应使用 `useMemo`）。
2. **逻辑与 UI 分离**: 复杂的业务逻辑建议抽离为自定义 Hook，保持组件代码简洁。
3. **副作用隔离**: 不要在渲染函数体中直接写副作用代码，务必包裹在 `useEffect` 中。
