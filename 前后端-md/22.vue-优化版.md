## 第一章：现代前端工程化与 Vue 3 启动

### 1.1 环境搭建：Vite vs Vue CLI
- **推荐方案 (Vite)**: `npm create vite@latest my-vue -- --template vue`
  - **原理**: **No-bundle ESM**。利用浏览器原生的 ES 模块加载，开发环境无需打包，实现秒级启动和极速热更新 (HMR)。
- **经典方案 (Vue CLI)**: `npm install -g @vue/cli` -> `vue create my-vue`
  - **原理**: 基于 **Webpack**。启动前必须先抓取并构建整个应用的模块树。

### 1.2 入口文件与挂载流程 (main.js)
```javascript
import { createApp } from 'vue'
import App from './App.vue'

// 1. 创建应用实例 (createApp 隔离了应用上下文)
const app = createApp(App)
// 2. 挂载到 HTML 中 id 为 app 的容器
app.mount('#app')
```

## 第二章：模板语法、组件通信与包裹

Vue 的核心是 **声明式渲染** 与 **组件化**。理解组件间的数据流动与内容包裹是构建复杂应用的基础。

### 2.1 属性识别的三类规则
1.  **静态属性**: 无前缀。引号内永远是 **纯字符串**。
    - `<el-form label-position="top">` -> "top" 是固定值。
2.  **动态绑定 (:)**: 属性名前加冒号。引号内是 **JS 表达式 (变量/函数/运算)**。
    - `<img :src="imageUrl">` -> 去寻找名为 `imageUrl` 的响应式变量。
3.  **指令模式 (v-)**: 以 `v-` 开头。引号内 **默认就是变量或表达式**。
    - `<div v-if="isLoggedIn">` -> 自动解析 `isLoggedIn` 的布尔值。

### 2.2 核心指令全集
- **插值表达式**: `{{ 变量名 }}`。展示响应式数据。
- **条件渲染**:
  - `v-if / v-else-if / v-else`: **物理操作**。不满足条件时 DOM 节点不被创建。
  - `v-show`: **CSS 操作**。始终创建节点，仅切换 `display: none`。
- **列表渲染 (v-for)**:
  - **语法**: `<li v-for="item in items" :key="item.id">`。
- **双向绑定 (v-model)**:
  - **本质**: 它是 `:value` 与 `@input` 的语法糖，实现了视图与状态的同步更新。

### 2.3 组件通信全家桶 (传参细节)

> **知识点补充：这些函数哪来的？**
> - **defineProps / defineEmits**: 属于“编译器宏”，Vue 内置，**不需要 import**，只能在 `<script setup>` 中使用。
> - **provide / inject / ref**: 属于组合式 API，必须 **手动 import** (从 'vue' 包中)。

#### **(1) 父传子 (Props)：向下传递指令**
- **场景**: 父组件告诉子组件该显示什么。
- **父组件如何传**: 像写 HTML 属性一样。
  - 静态值：`title="固定文字"`
  - 动态变量：`:count="num"` (必须加冒号)
- **代码示例**:
  ```html
  <!-- 父组件 Parent.vue -->
  <template>
    <Son title="今日任务" :count="100" />
  </template>

  <!-- 子组件 Son.vue -->
  <script setup>
  const props = defineProps({
    title: String,
    count: Number
  })
  </script>
  <template>
    <h1>{{ title }} - {{ count }}</h1>
  </template>
  ```

#### **(2) 子传父 (Emits)：向上传递信号**
- **场景**: 子组件发生了某些事（点按钮、输入完），通知父组件处理。
- **流程**: 子组件发信号 -> 父组件用 `@` 监听。
- **代码示例**:
  ```html
  <!-- 子组件 Son.vue -->
  <script setup>
  const emit = defineEmits(['buy']) // 1. 声明信号名
  const handleClick = () => emit('buy', 5) // 2. 发出信号并带上数据
  </script>

  <!-- 父组件 Parent.vue -->
  <template>
    <Son @buy="handleSonBuy" /> <!-- 3. 监听信号 -->
  </template>
  <script setup>
  const handleSonBuy = (num) => console.log('儿子买了', num) // 4. 接收处理
  </script>
  ```

#### **(3) 跨层级通信 (Provide / Inject)：广播模式**
- **场景**: 爷爷组件直接传给曾孙子，避开中间层的冗余传递（Props 钻取）。
- **代码示例**:
  ```javascript
  // 爷爷组件
  import { provide, ref } from 'vue'
  const money = ref(100)
  provide('family-money', money) // 挂出广播

  // 曾孙子组件
  import { inject } from 'vue'
  const money = inject('family-money') // 打开收音机
  ```

#### **(4) 内容包裹与定制 (Slot 插槽)**
- **默认插槽**: 子组件留个 `<slot />`，父组件在标签内填的内容会替换进去。
- **具名插槽**: 
  - 子组件：`<slot name="header" />`
  - 父组件：`<template #header> 内容 </template>` (使用 `#` 缩写)
- **作用域插槽 (进阶)**: 子组件把内部循环的数据传给父组件决定如何显示。
  - 子组件：`<slot :row="item" />`
  - 父组件：`<template #default="{ row }"> {{ row.name }} </template>` (解构获取数据)

## 第三章：Composition API 深度全解 (现代标准)

在 Vue 3 中，`<script setup>` 是目前官方最推荐的写法。它解决了逻辑碎片化问题，并提供了更强大的类型推断。

### 3.1 响应式核心：ref 与 reactive

#### **(1) ref() —— 万能的响应式引用**
- **用途**: 声明基本类型（string, number, boolean）或对象、数组的响应式。
- **语法细节**:
  - **JS/TS 中**: 必须通过 `.value` 访问或修改。
    ```javascript
    const count = ref(0);
    count.value++; // 修改
    ```
  - **Template 中**: 自动“解包”，无需写 `.value`。
- **进阶：DOM 引用**:
  - `const inputRef = ref(null)` -> `<template> <input ref="inputRef"> </template>`。

#### **(2) reactive() —— 深度响应式对象**
- **用途**: 声明**对象类型**的响应式代理（Object, Array, Map, Set）。
- **用法语法**:
  - **直接访问**: 不需要 `.value`，直接通过 `state.prop` 访问。
    ```javascript
    const state = reactive({ count: 0, list: [1, 2] });
    state.count++; // 直接改
    state.list.push(3); // 深度响应，数组操作也生效
    ```
- **三个核心限制 (避坑指南)**:
  1. **类型限制**: 只能用于引用类型（对象/数组），不能用于字符串、数字、布尔值。
  2. **替换丢失响应性**: 不能直接给整个对象赋值，否则会断开代理连接。
     - ❌ `state = { count: 1 }` (这样 state 就不再是响应式的了)
     - ✅ `state.count = 1` 或 `Object.assign(state, { count: 1 })`
  3. **解构丢失响应性**: 直接对 reactive 对象进行解构会使其变为普通变量。
     - ❌ `const { count } = state;` (count 变了页面不会动)
     - ✅ `const { count } = toRefs(state);` (使用 toRefs 保持引用)

#### **(3) ref vs reactive 对比总结**
| 特性 | ref | reactive |
| :--- | :--- | :--- |
| **数据类型** | **全能** (基本类型 + 引用类型) | **仅限对象类型** (Object/Array/Map/Set) |
| **访问方式** | JS 中需 `.value`，模板中不需要 | 任何地方都不需要 `.value` |
| **底层实现** | 基本类型用 `RefImpl` 类；对象用 `reactive()` | 基于原生 `Proxy` |
| **重新赋值** | 直接修改 `.value` 依然保持响应性 | 直接修改对象引用会**丢失响应性** |
| **推荐建议** | **官方推荐优先使用 `ref`**，因为它能处理所有情况且逻辑更清晰。 | 仅在需要组织大型关联数据对象时使用。 |

### 3.2 computed：计算属性 (智能缓存)
- **原理**: 基于响应式依赖进行缓存。只有当依赖的 `ref` 变化时，它才会重新计算。
- **缓存价值**: 
  - 只要组件刷新，**普通函数** 就会重跑（即使依赖没变）。
  - **Computed** 只有在依赖变了才重跑，节省昂贵的 CPU 算力。
- **语法**:
  ```javascript
  const double = computed(() => count.value * 2);
  ```

---

### 3.3 侦听器：watch 与 watchEffect

#### **(1) watch**
- **特性**: 懒执行（除非设置 `immediate: true`），可以访问 `oldValue`。
- **语法**:
  ```javascript
  watch(count, (newVal, oldVal) => {
    console.log('新值:', newVal, '旧值:', oldVal);
  }, { immediate: true, deep: true });
  ```

#### **(2) watchEffect**
- **特性**: **立即执行**，且自动追踪函数内部使用的所有响应式变量。
- **场景**: 当逻辑中依赖多个变量，且不需要关心旧值时，代码更简洁。
  ```javascript
  watchEffect(() => console.log('当前计数:', count.value));
  ```

---

### 3.4 生命周期钩子
在组合式 API 中，生命周期通过导入函数使用：
- **onMounted**: DOM 挂载完成。适合发请求、初始化图表。
- **onUpdated**: 组件 DOM 更新后。
- **onUnmounted**: 组件销毁。适合清理定时器、解绑事件。
- **对比 React**: Vue 的钩子是基于组件状态机的确切时刻；React 的 `useEffect` 则是将“挂载”与“状态同步”耦合在一起。

---

### 3.5 逻辑复用：组合式函数 (Composables)

#### **(1) 本质与原理**
组合式函数是 Vue 3 真正的核心威力，它类似于 React Hooks。通过利用 Composition API 来封装和复用**有状态逻辑**。

#### **(2) 编写规范与语法**
1. **命名**: 约定以 `use` 开头（如 `useMouse`）。
2. **逻辑封装**: 内部可以使用 `ref`、`onMounted` 等所有组合式 API。
3. **返回数据**: 必须返回一个对象，供组件解构使用。

**示例：封装一个倒计时逻辑 (`useCountDown.js`)**
```javascript
import { ref, onUnmounted } from 'vue';

export function useCountDown(initialTime) {
  const time = ref(initialTime);
  const timer = setInterval(() => {
    if (time.value > 0) time.value--;
    else clearInterval(timer);
  }, 1000);

  onUnmounted(() => clearInterval(timer));

  return { time };
}
```

## 第四章：Vue Router 4.x 路由管理与实战

### 4.1 核心原理与模式
- **Web History**: 基于 HTML5 History API，路径干净（如 `/home`）。
- **Web Hash**: 基于 URL 哈希，兼容性强（如 `/#/home`）。

### 4.2 路由跳转与导航 (Usage)

#### **(1) 声明式导航**
- `<router-link to="/home">首页</router-link>`
- `<router-link :to="{ name: 'user', params: { id: 123 }}">用户</router-link>`

#### **(2) 编程式导航 (useRouter)**
```javascript
import { useRouter } from 'vue-router';
const router = useRouter();

// 跳转到指定路径
router.push('/login');
// 跳转并传参 (query)
router.push({ path: '/search', query: { keyword: 'vue' } });
// 替换当前记录 (不产生历史回退)
router.replace('/welcome');
// 后退/前进
router.back(); // 等价于 router.go(-1)
```

### 4.3 路由信息获取 (useRoute)
```javascript
import { useRoute } from 'vue-router';
const route = useRoute();

console.log(route.path);      // 当前路径: /user/123
console.log(route.params.id); // 路径参数: 123
console.log(route.query.q);   // 查询参数: ?q=search
console.log(route.fullPath);  // 完整路径
console.log(route.name);      // 路由名称
console.log(route.meta);      // 路由元信息 (如权限标记)
```

### 4.4 路由守卫与重定向
- **重定向 (Redirect)**: `{ path: '/', redirect: '/home' }`
- **全局守卫**:
  ```javascript
  router.beforeEach((to, from, next) => {
    // to.meta.requiresAuth 可以在路由配置中自定义
    if (to.meta.requiresAuth && !isLoggedIn) {
      next('/login'); // 强制拦截跳转
    } else {
      next(); // 放行
    }
  });
  ```

## 第五章：Pinia：极简状态管理

### 5.1 创建 Store (defineStore)
```javascript
export const useUserStore = defineStore('user', () => {
  const currentUserId = ref('1001'); // State
  const setCurrentUser = (id) => currentUserId.value = id; // Action
  const doubleId = computed(() => currentUserId.value * 2); // Getter
  return { currentUserId, setCurrentUser, doubleId };
})
```

### 5.2 组件内使用
```javascript
const userStore = useUserStore();
console.log(userStore.currentUserId); // 读取
userStore.setCurrentUser('1002'); // 修改，所有监听该 Store 的组件自动更新
```

## 第七章：Vue vs React 深度对比

### 7.1 渲染模式与更新机制
这是两者最本质的区别：

- **React: “全量复印”模式**
  - **机制**: 只要父组件 State 变了，默认会重跑该组件及其下所有子组件。
  - **副作用**: 容易产生无效重绘，必须手动用 `memo`, `useMemo` 来“保命”。
  - **心智负担**: 开发者需要精准控制依赖项，否则会产生闭包陷阱或性能崩溃。

- **Vue: “精准监控”模式**
  - **机制**: 响应式系统会自动建立“依赖地图”。Vue 知道哪个变量被哪个组件使用了。
  - **优势**: 变量 A 变了，只会触发真正使用了 A 的组件更新。子组件如果没用到这个变量，绝对不会重跑。
  - **心智负担**: 极低。框架帮你完成了绝大部分性能优化。

### 7.2 技术特性对比表

| 维度 | Vue 3 | React |
| :--- | :--- | :--- |
| **底层驱动** | **响应式 (Proxy)**: 自动追踪依赖，局部精准更新 | **不可变性 (Immutable)**: 手动触发更新，全量 Diff |
| **上手门槛** | 低（HTML 模板符合直觉） | 高（需要深刻理解 JSX 与 JS 闭包） |
| **计算与缓存** | `computed` (自动依赖收集与缓存) | `useMemo` (手动声明依赖项) |
| **状态管理** | Pinia (官方全家桶，逻辑统一) | Zustand/Redux (社区方案，选型多) |
| **性能优化** | **全自动**: 编译时进行静态提升、PatchFlag 标记 | **半手动**: 需使用 `memo`, `useCallback` 防止无效重绘 |

## 第九章：Vue 3 + TypeScript 增强

TypeScript (TS) 的本质是给代码加个 **“说明书”**，实现了 **代码自文档化**。

### 9.1 为什么要用 TS？
1.  **防呆 (报错提示)**: 规定传数字却传了字符串，写代码时 VSCode 就会红线报错，而不是运行后才崩溃。
2.  **自动提示**: 输入 `user.` 后，VSCode 自动列出所有属性，不用去猜或翻文档。

### 9.2 Props 类型标注
推荐使用 **基于类型的声明**（仅限 `<script setup>`）。
```typescript
interface Props {
  title: string;  // 规定 title 必须是字符串
  count?: number; // 规定 count 必须是数字，? 表示可选
}

// 使用泛型 <Props> 给组件打“标签”
// 这样做之后，如果父组件传错类型，编辑器会直接报错
const props = defineProps<Props>();
```

### 9.3 Ref 与 Reactive 类型标注
1. **Ref**: 使用泛型 `ref<T>()`。
   ```typescript
   const count = ref<number>(0);
   const user = ref<User | null>(null); // user 初始为 null，以后是 User 对象
   ```
2. **Reactive**: 通过定义 `interface` 约束。
   ```typescript
   interface State {
     list: string[];
     loading: boolean;
   }
   const state = reactive<State>({ list: [], loading: false });
   ```
