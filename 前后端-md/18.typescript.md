# TypeScript 核心笔记 (优化版)

> 本笔记将 TS 知识点划分为：**环境配置**、**基础类型**、**对象与复用**、**进阶高级类型**、**工程实践** 五大模块，方便快速复习与查阅。

***

## 一、 环境配置与工具 (Environment)

### 1. 安装与运行

- **全局安装**：`npm i -g typescript`
- **传统运行**：
  - 编译：`tsc 文件名.ts` (生成同名 `.js`)
  - 执行：`node 文件名.js`
- **便捷运行**：`npm i -g ts-node`
  - 直接运行：`ts-node 文件名.ts` (内部编译不落地 js 文件)

***

## 二、 基础类型系统 (The Basics)

### 1. 类型注解 (Type Annotation)

- **语法**：`let 变量名: 类型 = 值`
- **基本类型**：`number`, `string`, `boolean`, `null`, `undefined`

### 2. 联合类型与自定义类型

- **联合类型**：`(类型1 | 类型2)`
- **类型别名 (type)**：`type MyType = string | number` (用于复用复杂类型)

### 3. 特殊类型

- **字面量类型**：指定变量的值必须等于某个具体的值（字符串/数字/对象）。
- **any 类型**：**慎用！** 绕过所有类型检查。
- **void 类型**：用于函数，表示没有返回值。

### 4. 容器类型 (数组与元组)

- **数组**：
  - `类型[]` (推荐)：如 `number[]`，表示“这是一个全是数字的数组”。
  - `Array<类型>`：泛型写法，如 `Array<string>`。
  - **说明**：数组类型限制了成员的类型，一旦声明，存入不同类型的数据会报错。
- **元组 (Tuple)**：
  - `[类型1, 类型2]`：如 `[string, number]`。
  - **说明**：一种特殊的数组，**长度固定**且**每一项类型必须对应**。常用于函数返回多个不同类型的值（如 React 的 `useState`）。

### 5. 函数类型

- **核心组成**：参数类型 + 返回值类型。
- **声明方式**：
  - **普通函数**：`function add(a: number, b: number): number { return a + b }`
  - **箭头函数**：`const add = (a: number, b: number): number => a + b`
  - **函数表达式别名**：`const add: (a: number, b: number) => number = (a, b) => a + b`
- **可选参数**：`参数名?`。**注意**：可选参数必须放在必选参数之后。
- **说明**：TS 强制检查调用函数时传入的参数个数和类型是否匹配。

## 三、 对象、接口与类 (Objects & Reusability)

### 1. 对象与接口 (Interface)

- **对象注解**：直接在变量名后写结构，如 `let obj: { name: string; age?: number }`。
- **接口声明**：
  ```typescript
  interface Person {
    name: string;
    sayHi(): void;
  }
  ```
- **接口继承 (extends)**：
  - **用途**：从已有的接口派生新接口，实现属性复用。
  - **举例**：
    ```typescript
    interface Animal { name: string }
    interface Dog extends Animal { bark(): void } 
    // Dog 自动拥有了 name 属性和自己的 bark 方法
    ```
- **声明合并 (Declaration Merging)**：
  - **特点**：允许定义多个同名的 `interface`，TS 会自动将它们合并。
  - **场景**：常用于扩展第三方库的类型定义。
  - **示例**：
    ```typescript
    interface User { name: string }
    interface User { age: number }
    // 最终 User 的结构为 { name: string; age: number }
    ```
- **对比：Interface vs Type**
  - `interface`：专为对象设计，支持 **extends (继承)**，支持 **同名声明合并** (允许重复定义)。
  - `type`：更灵活，可用于任何类型（联合、原始类型等），支持 **& (交叉)**，**不支持重复定义**。

### 2. 类 (Class)

- **基本用法**：`class Animal {}`，`const dog = new Animal()`。
- **可见性修饰符**：
  - **说明**：用于控制类成员（属性/方法）在哪里可以被访问，增强代码的安全性（封装性）。

| 修饰符           | 类内部访问 | 子类访问 | 实例访问 | 详细说明                                  |
| :------------ | :---: | :--: | :--: | :------------------------------------ |
| **public**    |   ✅   |   ✅  |   ✅  | **公开的**：默认值，哪里都能用。                    |
| **protected** |   ✅   |   ✅  |   ❌  | **受保护的**：只能在“家族”内部传阅（父类和子类），外部实例看不见。  |
| **private**   |   ✅   |   ❌  |   ❌  | **私有的**：只有在定义它的那个类内部能用，儿子（子类）和外人都拿不到。 |

- **实现接口**：`class Dog implements AnimalInterface {}` (类必须拥有接口定义的所有外观)

## 四、 进阶高级类型 (Advanced Types)

### 1. 泛型 (Generics) - 类型变量

- **是什么**：泛型就像是“类型的形参”。你可以把类型当成变量传进去。
- **为什么要用**：
  - 如果一个函数能处理多种类型（比如返回传入的任何东西），用 `any` 会丢失类型检查。
  - 泛型能让你在**定义时不确定类型**，在**调用时才决定类型**，同时保持类型安全。
- **泛型函数**：`function id<T>(value: T): T { return value }`
  - `T` 是一个占位符，调用 `id<number>(10)` 时，`T` 就变成了 `number`。
- **泛型接口**：
  - **用途**：定义一个通用的接口结构，但其中的某些属性类型由调用者决定。
  - **示例**：
    ```typescript
    interface Response<T> {
      code: number;
      data: T; // data 的具体类型由 T 决定
    }
    // 使用时：
    const userRes: Response<{ name: string }> = { code: 200, data: { name: 'zs' } };
    ```
- **泛型类**：
  - **示例**：
    ```typescript
    class GenericNumber<T> {
      zeroValue: T;
      add: (x: T, y: T) => T;
    }
    const myNum = new GenericNumber<number>();
    myNum.zeroValue = 0;
    ```
- **约束 (extends)**：`fn<T extends { length: number }>`
  - **场景**：当你需要操作 `T` 上的某个属性（如 `.length`）时，必须先保证 `T` 真的有这个属性。

### 2. 类型推论、断言与 typeof

- **类型推论**：
  - **生效时机**：1. 声明变量并立即初始化时；2. 函数返回结果时。
  - **意义**：能不写类型就不写，让代码更简洁。
- **类型断言 (as)**：
  - **用途**：手动覆盖 TS 的类型推断。当你比 TS 更清楚某个值的具体类型时使用。
  - **核心场景**：从 DOM 获取元素。TS 只知道它是一个 `Element`，但你确定它是一个 `HTMLAnchorElement`（超链接），这样你才能访问 `.href` 属性。
  - **示例**：`const myLink = document.getElementById('link') as HTMLAnchorElement;`
- **typeof**：
  - **用途**：在“类型上下文”中引用一个**变量或属性的类型**。
  - **示例**：
    ```typescript
    let p = { x: 10, y: 20 };
    // 我想要一个和 p 结构一模一样的类型，不需要重新定义 interface
    type Point = typeof p; // Point 现在就是 { x: number, y: number }
    ```

### 3. 索引签名

- **场景**：当你不知道对象会有哪些具体的键，只知道键和值的类型规律时。
- **对象索引**：`interface Map { [key: string]: number }` (可以有任意个 string 类型的键，值必须是数字)
- **数组索引**：`interface MyArr { [index: number]: string }` (模拟数组)

### 4. 内置工具类型 (Utility Types)

- **用途**：基于一个已有的对象类型，快速“改造”出一种新类型。

| 工具                | 功能说明        | 为什么要用                            | 代码示例                                          |
| :---------------- | :---------- | :------------------------------- | :-------------------------------------------- |
| **Partial<T>**    | 属性变 **可选**  | 比如修改用户信息时，只需要传部分字段。              | `type UpdateUser = Partial<User>`             |
| **Readonly<T>**   | 属性变 **只读**  | 防止配置对象被意外修改。                     | `const config: Readonly<Conf> = {...}`        |
| **Pick\<T, K>**   | **挑选** 部分属性 | 只需要大对象中的几个字段（如只要用户的 ID 和 Name）。  | `type UserBrief = Pick<User, 'id' \| 'name'>` |
| **Record\<K, T>** | **批量创建**    | 快速定义一个键值对结构，如所有键都是 string，值都是数字。 | `type Scores = Record<string, number>`        |

### 5. 映射类型 (Mapping)

- **核心逻辑**：基于旧类型，通过“遍历”产生新类型。
- **in**：遍历联合类型。
  - `type Keys = 'a' | 'b'; type Obj = { [K in Keys]: number }` -> `{ a: number, b: number }`
- **keyof**：获取对象类型的所有键名。
  - `keyof { name: string, age: number }` -> `'name' | 'age'`
- **索引查询**：`T[K]`
  - **用途**：获取对象类型中某个键对应的**具体类型**。
  - **示例**：`Person['age']` 得到的就是 `number` 类型。

## 五、 工程实践 (Engineering)

### 1. 模块化：类型的导入与导出

- **导出 (Export)**：
  ```typescript
  // types.ts
  export interface User { id: number; name: string }
  export type ID = number | string;
  ```
- **导入 (Import)**：
  ```typescript
  // main.ts
  import { User, ID } from './types';
  // 推荐写法（明确告知这是在导入类型，构建时会被完全移除）：
  import type { User } from './types';
  ```

### 2. 类型声明文件 (.d.ts)

- **作用**：仅提供类型声明，不产生 JS 代码。
- **获取方式**：
  1. 包自带（查看 `package.json` 的 `typings`）。
  2. 社区提供：`npm i -D @types/包名`。
- **declare 关键字**：用于为非 TS 环境（如直接引入的 JS 脚本）手动声明全局变量、函数或模块。
